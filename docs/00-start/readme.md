## Naming Conventions

All documentation follows kebab-case naming:
- `documentation-index.md` (not DOCUMENTATION.md)
- `changelog.md` (not CHANGELOG.md)
- `readme.md` (not README.md)

This ensures consistency across all documentation files.

## Documentation Principles

1. **Single Source of Truth** - Each topic has one authoritative document
2. **Link > Duplicate** - Reference other docs instead of copying content
3. **Scannable > Comprehensive** - Tables, bullets, clear headers
4. **Practical > Theoretical** - Code examples and real use cases
5. **Current > Complete** - Keep docs updated as code changes
